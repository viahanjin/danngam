# Danngam MVP 패턴 테스트 계획

## 문서 정보

- **버전**: 1.0
- **작성일**: 2026-02-14
- **최종 수정일**: 2026-02-14
- **대상 독자**: QA 테스터, 기획자
- **목적**: Danngam 프로젝트의 기능 및 비기능 테스트 계획 및 체크리스트

---

## 1. 테스트 전략

### 1.1 테스트 범위

#### 기능 테스트 (Functional Testing)
- 사용자 인증 (회원가입, 로그인, 로그아웃)
- 장비 검색 및 필터링 (이동형/고정형)
- 장비 상세 조회
- 예약 프로세스
- 결제 및 환불
- 리뷰 및 평가
- 프로필 관리

#### 비기능 테스트 (Non-Functional Testing)
- 성능 (응답 시간, 로드 시간)
- 보안 (HTTPS, 암호화, 인증)
- 호환성 (다양한 디바이스/OS)
- 접근성 (폰트 크기, 버튼 크기)

### 1.2 테스트 환경

#### Android 환경
```
OS: Android 11, 12, 13, 14
디바이스: Samsung Galaxy S21/S23, Google Pixel 6/7
해상도: 1440x3200, 1080x2400
화면: 6.2" ~ 6.7"
```

#### iOS 환경
```
OS: iOS 15, 16, 17
디바이스: iPhone 13, 14, 15
해상도: 1170x2532, 1242x2688
화면: 6.1" ~ 6.7"
```

#### 네트워크 환경
```
- Wi-Fi (5GHz, 2.4GHz)
- 4G LTE
- 5G (선택사항)
- 오프라인 (캐시 확인)
```

### 1.3 테스트 레벨

| 레벨 | 범위 | 담당자 | 도구 |
|------|------|--------|------|
| **Unit** | 개별 함수/클래스 | 개발자 | Flutter Test |
| **Widget** | 개별 위젯 | 개발자 | Flutter Test |
| **Integration** | 여러 모듈의 상호작용 | QA | Flutter Driver |
| **E2E** | 전체 사용자 플로우 | QA | 수동 테스트 |

## 2. 기능 테스트 케이스

### 2.1 TC-001~010: 로그인 및 회원가입

#### TC-001: 전화번호 입력 및 OTP 발송

```
[테스트 케이스 ID] TC-001
[제목] 전화번호 입력 후 OTP 발송
[선행 조건] 로그인 화면 진입
[테스트 단계]
1. '회원가입' 버튼 클릭
2. 휴대폰 번호 입력 (예: 010-1234-5678)
3. '인증번호 발송' 버튼 클릭
[예상 결과]
- 입력한 번호로 OTP 발송 확인 (문자 수신)
- 화면에 'OTP 입력' 필드 표시
- 재발송 버튼 활성화
- 타이머 시작 (3분)
[통과 기준] OTP가 정상적으로 발송되고 입력 필드가 표시됨
[우선순위] High
```

#### TC-002: OTP 확인 및 인증 성공

```
[테스트 케이스 ID] TC-002
[제목] OTP 확인 성공
[선행 조건] TC-001 완료 (OTP 발송됨)
[테스트 단계]
1. 수신한 OTP 입력 (6자리 숫자)
2. '인증' 버튼 클릭
[예상 결과]
- 5초 내 인증 성공 메시지
- 다음 단계 화면으로 이동 (약관 동의)
[통과 기준] OTP 인증 성공 및 화면 전환
[우선순위] High
```
