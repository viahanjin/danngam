## 1. SplashScreen (시작 화면)

### 레이아웃

```
┌─────────────────────────────────┐
│                                 │
│                                 │
│          단감 로고               │
│        (아이콘 큼)              │
│                                 │
│       로딩 인디케이터            │
│       (스피너)                  │

```

### 요소 정의

| 요소 | 크기 | 설명 |
|------|------|------|
| 로고 | 100x100px | 화면 중앙, 둥근 모양 아이콘 |
| 로딩 애니메이션 | - | 회전 스피너 |

### 요소 정의

#### 헤더
- 좌: ← 뒤로 버튼
- 중: "로그인" 타이틀
- 우: 없음

#### 전화번호 입력 (상태 1)
- InputField 너비: 100%
- 높이: 56px
- 테두리: 1px #E5E7EB
- 패딩: 16px
- 플레이스홀더: "01012345678"
- 타입: tel (숫자만 입력 가능)

#### OTP 입력 (상태 2)
- InputField 너비: 100%
- 높이: 56px
- 타입: numeric (6자리 숫자)
- 자동 포커싱

#### 체크박스
- 크기: 24x24px
- 라벨: 클릭 가능 (라벨 클릭 시 체크박스 토글)
- 링크: > 아이콘 (약관 상세 오픈)

#### 버튼
- [OTP 전송]: 너비 100%, 높이 56px, Primary Color
- [인증 완료]: 너비 100%, 높이 56px, Primary Color
- [OTP 재전송]: 텍스트 버튼, 파란색
- [모두 동의]: 약관 모두 동의 후 활성화

### 사용자 인터랙션

**상태 1:**
- 전화번호 입력 → [OTP 전송] 버튼 활성화
- [OTP 전송] 클릭 → 상태 2로 변경
- 약관 동의 필수 (체크박스 2개)

**상태 2:**
- OTP 6자리 입력 → [인증 완료] 버튼 활성화
- 타이머: 3분 (카운트다운)
- [인증 완료] 클릭 → MainScreen으로 이동
- [OTP 재전송] 클릭 → 타이머 리셋, 새 OTP 발송

**상태 3:**
- 타이머 만료 → 상태 3으로 변경
- [OTP 재전송] 클릭 → 상태 2로 복귀

### 상태별 변형

| 상태 | 설명 |
|------|------|
| 입력 중 | 커서 표시, 입력 중 버튼 disabled |
| OTP 전송 중 | [OTP 전송] disabled + 로딩 스피너 |
| OTP 입력 대기 | 타이머 카운트다운, [인증 완료] disabled |

| 만료됨 | ⚠️ 에러 메시지, [OTP 재전송] 버튼만 활성 |

## 개발 시 주의사항

1. **이동형/고정형 구분**: 탭 클릭 시 API 호출 파라미터에 `type=MOBILE` 또는 `type=FIXED` 포함
2. **무한 스크롤**: ListView.builder로 구현하여 성능 최적화
3. **캐싱**: 이미지는 cached_network_image 사용
4. **상태 관리**: Presenter에서 리스트 상태 관리 (로딩, 데이터, 에러)
5. **애니메이션**: 탭 전환 시 Fade 또는 Slide 효과
